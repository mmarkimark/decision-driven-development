# Rule Hierarchy

This defines the authority order of rules in the methodology.

| Level | Authority | Scope |
|------|-----------|-------|
| 1️⃣ | Decisions | Architecture and system model |
| 2️⃣ | Architecture Rules | DDD boundaries of the method |
| 3️⃣ | Domain Rules | Derivation, bugs, domain events |
| 4️⃣ | Workflow Rules | Git flow, Fast-Lanes, PR rules |
| 5️⃣ | AI Constraints | How AI agents are allowed to operate |

Higher levels override lower ones.
